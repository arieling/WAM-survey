## One-Sentence Summary

> π₀.7 is a ~5B-parameter generalist robot foundation model that achieves emergent cross-embodiment generalization and language instruction following through diverse context conditioning — incorporating subtask instructions, subgoal images (from a BAGEL 14B world model), and episode metadata (speed/quality/mistakes) — trained on a heterogeneous mix of demonstrations, autonomous evaluation data, and non-robot sources.

---

## Core Contributions

1. **Diverse Context Conditioning**: A rich multi-component prompt structure combining subtask instructions, subgoal images, episode metadata (speed, quality, mistake labels), and control mode — enabling a single policy to steer behavior across diverse tasks, embodiments, and quality levels.
2. **World-Model-Assisted Subgoal Generation**: BAGEL (14B mixture-of-transformers) generates subgoal images conditioned on observations and subtask instructions via flow matching, providing visual plan guidance that substantially improves performance on complex multi-stage tasks.
3. **Mixed-Quality Data Training with Metadata Conditioning**: By conditioning on episode quality and mistake labels, the model learns to leverage suboptimal demonstrations (failure episodes, lower-quality data) — enabling training on 100% of available data without degradation.
4. **Emergent Capabilities**: Zero-shot cross-embodiment generalization and long-horizon language coaching (LLM-assisted human coaching → autonomous policy transfer) emerge from the diverse conditioning scheme without task-specific fine-tuning.

---

## Problem Background

### Problem to Solve
Generalist robot policies must handle diverse tasks, embodiments, and instruction types simultaneously. Existing models (π₀.₅, π₀.₆) require task-specific fine-tuning and struggle with: (1) complex multi-stage dexterous tasks, (2) zero-shot cross-embodiment transfer, (3) language instruction following in unseen environments, and (4) leveraging suboptimal training data effectively.

### Limitations of Existing Methods
- π₀.₅ / π₀.₆: limited multimodal conditioning; suboptimal performance on complex instruction following; no mixed-quality data mechanism.
- RL-finetuned specialists: require task-specific fine-tuning; not generalizable.
- Standard VLA (language-conditioned): ignores subgoal visual guidance, episode quality metadata, and speed/control mode information.

### Motivation
Rich context conditioning expands the information available to the policy at inference time, enabling it to "steer" toward desired behaviors — matching a specialist RL policy's performance out-of-the-box by providing relevant task context (subgoal images for complex tasks, quality metadata for mixed data training, speed metadata for throughput optimization).

## Method Details

### Model Architecture

π₀.7 is a ~**5B-parameter VLA** with flow-matching action generation:

1. **VLM Backbone**: 4B parameters (Gemma3-based); processes multimodal context via block-causal masking
2. **Vision Encoder**: 400M parameters (Gemma3-based); MEM-style video history compression for multi-camera inputs
3. **Action Expert**: 860M parameters; transformer with flow matching, adaptive RMSNorm for timestep injection

**Block-Causal Masking**: observation + subgoal tokens use bidirectional attention; text tokens use causal attention.

### Core Modules

#### Module 1: Multi-Component Context Conditioning

**Design Motivation**: Different task aspects require different types of guidance — language for semantic understanding, subgoal images for spatial planning, metadata for quality control, control mode for embodiment adaptation.

**Context Components**:
- **Subtask instructions** $\hat{\ell}_t$: intermediate semantic descriptions of current sub-goal (30% dropout when subgoal image present)
- **Subgoal images** $g_t$: multi-view future state images from world model or ground truth; included in 25% of training examples
  - 25% from real end-of-segment frames, 75% uniformly sampled 0–4 seconds ahead
- **Episode metadata** $m$: speed (discretized at 500-step intervals), quality (1–5 scale), mistake labels; 15% full dropout, 5% per-component dropout
- **Control mode**: joint vs. end-effector designation

#### Module 2: World Model (BAGEL-based Subgoal Generator)

**Design Motivation**: Subgoal images provide visual spatial plans that complement language instructions, enabling complex multi-stage task execution. A learned world model allows subgoal generation at inference time without human annotation.

**Implementation**:
- Base model: BAGEL (14B mixture-of-transformers; image understanding + editing + generation)
- The BAGEL world model is trained to generate subgoal images via a conditional flow-matching objective. Given dataset $D_g$ of (observation, subtask instruction, metadata, target subgoal) tuples, the training objective maximizes:

$$\max_\psi \mathbb E_{D_g}[\mathcal L_{CFM}(g_t^*, g_\psi(o_t, \hat{\ell}_t, m))]$$

where $g_t^*$ is the ground-truth subgoal image, $o_t$ is the current observation, $\hat{\ell}_t$ is the subtask instruction, and $m$ is episode metadata. This trains BAGEL to generate the predicted future visual state conditioned on the current context.

#### Module 3: Action Expert (Flow Matching)

**Design Motivation**: Flow matching provides stable, high-quality action generation; 5-step denoising at inference enables real-time control.

**Implementation**:
- 860M transformer parameters
- Generates 50-step action chunks
- Adaptive RMSNorm injects diffusion timestep information

The primary policy learning objective is maximum-likelihood behavioral cloning over action chunks conditioned on observation history and the full multi-component context $C_t = (\hat{\ell}_t, g_t, m, \text{control mode})$:

$$\max_\theta \mathbb E_D\left[\log \pi_\theta(a_{t:t+H} \mid o_{t-T:t}, C_t)\right]$$

where $H=50$ is the action chunk horizon and $T$ is the observation history length.

At inference, classifier-free guidance can be applied to amplify the conditioning signal:

$$\tilde a_{t:t+H} = (1-\beta) \cdot \pi_\theta(\cdot \mid o, \emptyset) + \beta \cdot \pi_\theta(\cdot \mid o, C_t)$$

With $\beta > 1$, the guidance linearly amplifies the conditioned policy prediction relative to the unconditional baseline, yielding sharper, more task-directed behavior. $\beta \in \{1.3, 1.7, 2.2\}$ is tuned per task.

- 5 denoising steps at inference (fast)
- Execution of 15–25 steps per chunk

#### Module 4: Multi-Camera Vision Encoding

**Design Motivation**: Rich visual context from multiple cameras reduces ambiguity in complex manipulation scenes; temporal history enables motion understanding.

**Implementation**:
- Up to four cameras with six history frames each
- Frames resized to 448×448 pixels
- History frame sampling: 1-second stride, 0.3 dropout probability
- MEM-style video compression in vision encoder

### Implementation Details

- **Total Parameters**: ~5B (4B VLM + 400M vision + 860M action expert)
- **World Model**: BAGEL, 14B mixture-of-transformers
- **Action Chunk Size**: 50 steps; execute 15–25 per chunk
- **Denoising Steps (Inference)**: 5 steps
- **Camera Resolution**: 448×448 pixels, up to 4 cameras × 6 history frames
- **History Sampling**: 1-second stride, 0.3 dropout
- **Subgoal Image Frequency**: 25% of training examples
- **Metadata Dropout**: 15% full / 5% per-component
- **Classifier-Free Guidance**: $\beta \in \{1.3, 1.7, 2.2\}$
- **Control Frequency**: 20–50 Hz
- **Robot Platforms**: Bimanual mobile manipulator, BiPi, UR5e

### Out-of-the-Box Performance vs. Specialists

Four context components — subtask instructions (language), subgoal images (world-model or ground-truth future frames), episode metadata (speed/quality/mistakes), and control mode — are injected into the VLM backbone as multimodal tokens. This rich conditioning enables π₀.7 to match RL-finetuned specialists without any task-specific fine-tuning:

| Task | RL-finetuned Specialist | π₀.₇ |
|------|------------------------|-------|
| Espresso making | ~comparable | matches |
| Box building | ~comparable | matches/exceeds |
| Laundry folding | comparable | matches |

### Zero-Shot Cross-Embodiment (UR5e Shirt Folding)

The model demonstrates emergent cross-embodiment generalization: zero-shot transfer to UR5e (not seen during training) nearly matches expert human teleoperators:

| Method | Task Progress | Success Rate |
|--------|--------------|-------------|
| Experienced human teleoperator | 90.9% | 80.6% |
| **π₀.7** | **85.6%** | **80%** |

### Ablation — Mixed-Quality Data (Metadata Conditioning)

The metadata conditioning mechanism directly enables training on heterogeneous data quality. Without metadata labels, adding lower-quality data degrades performance; with metadata, the model continuously improves as more data is included:

| Training Data | With Metadata | Without Metadata |
|--------------|--------------|-----------------|
| 30% quality data | Improves | Degrades |
| 100% quality data | Best | Degrades from 30% |

### Task Diversity Ablation

The most diverse 20% of training data drives emergent generalization to unseen tasks — random data removal has minimal effect:

| Configuration | Unseen Short-Horizon Task Performance |
|--------------|--------------------------------------|
| Full π₀.7 | Best |
| w/o most diverse 20% of data | Substantially degraded |
| w/o random 20% of data | Minimal impact |

## Critical Analysis

### Strengths
1. Diverse context conditioning unlocks emergent capabilities (zero-shot cross-embodiment, language coaching) without task-specific fine-tuning.
2. Mixed-quality data utilization via metadata conditioning is highly practical — enables training on all available data, including failures and suboptimal demonstrations.
3. World-model-generated subgoal images provide visual planning guidance at inference time without human annotation.

### Limitations
1. **Model scale**: ~5B parameters + 14B world model is computationally expensive for deployment.
2. **World model quality dependency**: Subgoal generation quality is bottlenecked by BAGEL's accuracy — errors propagate to policy.
3. **Annotation cost**: Subtask instruction and episode metadata annotation require substantial human effort.
4. **Inference latency**: 5-step denoising + world model inference may limit control frequency in latency-sensitive scenarios.

### Potential Improvements
1. Distillation to smaller student model preserving emergent capabilities.
2. Online world model adaptation from deployment feedback.
3. Automated metadata annotation from robot sensors (force/torque for quality, encoder for speed).
